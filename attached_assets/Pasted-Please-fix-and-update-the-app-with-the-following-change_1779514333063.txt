1. Fix day navigation
The navigation between days does not work correctly.
When I click several days back, I cannot go forward day by day again.
Please fix the logic so the user can freely navigate backward and forward between all existing days.

2. Rename “Ideas” to “Recipes”
Rename the “Ideas” button and section everywhere in the app to “Recipes”.

3. Recipes generation logic
The “Recipes” section should generate one full daily recipe plan:
- Breakfast
- Lunch
- Dinner
- Snack

All meals together must fit within the user’s daily calorie limit.

The user should be able to:
- Regenerate the full day plan
- Regenerate only one meal separately: breakfast, lunch, dinner, or snack
- Add one single meal to their log
- Add the full suggested day to their log

The meals should make sense together as one coherent daily menu, not random disconnected dishes.

4. Save logs correctly after republishing
There is a serious issue after republishing the app: sometimes old logged days are not saved.
This happens especially when part of the data was entered from mobile and part from desktop. Also when log in/log out.

Please fix persistence and synchronization.
After every deployment / republish, all previously entered logs must still be available.
The user should not lose previous days or logs.

Make sure data is stored consistently and does not depend on the device used.

7. Remove “End of record”

8. Add Log out on mobile
In the mobile version, there is no Log out option.
